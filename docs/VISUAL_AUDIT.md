# Визуальный аудит «АвтоСделка»

## 1. Иерархия и типографика

**Проблемы:**
- Плоская типографическая шкала: `text-sm`, `text-lg`, `text-2xl` — недостаточно ступеней и ритма.
- H1 на listing: `text-2xl` — слабый акцент для hero-заголовка.
- Секционные заголовки (`text-lg`, `text-xl`) визуально сливаются.
- Inter везде — надёжно, но без характерной identity для premium-бренда.
- Не заданы `line-height`/`tracking` для заголовков.

**Рекомендации:**
- Усилить типографическую шкалу: добавить `text-3xl`/`4xl` для hero.
- Использовать `font-semibold`/`font-bold` консистентно по уровням.
- Добавить `tracking-tight` для крупных заголовков.

---

## 2. Отступы и ритм

**Проблемы:**
- Нерегулярные отступы: `py-6`, `py-8`, `mb-4`, `gap-4`, `gap-6` — нет единой сетки 4/8px.
- Search panel: `py-6` + `mb-4` + `gap-3` + `p-5` — визуальный шум.
- Карточки: `p-3` — плотно; listing blocks: `p-4` — разный ритм.
- `space-y-4` vs `gap-6` — несогласованность между секциями.

**Рекомендации:**
- Базовый ритм 8px: `4, 6, 8, 10, 12, 16, 20, 24`.
- Секции: `py-10`/`py-12` для крупных блоков.
- Внутри карточек: `p-5` как базовый padding.

---

## 3. Сетка и композиция

**Проблемы:**
- `max-w-7xl px-4` — хорошо, но внутренние grid различаются.
- Recommendations: `grid-cols-4` — карточки выглядят сжато на 1280px.
- Popular brands: `gap-2` — слишком плотно для premium.
- Listing: `lg:col-span-5 | 4 | 3` — пропорции ок, но визуальных разделителей нет.

**Рекомендации:**
- Добавить контейнерные классы для единообразия.
- Карточки: более щедрые отступы между карточками.
- Listing: визуальное разделение колонок (border/divider при необходимости).

---

## 4. Карточки автомобилей

**Проблемы:**
- Минималистичная карточка: только border, нет depth/shadow.
- Бейджи: `rounded-full` с `bg-success/90` — читаемо, но визуально простые.
- Hover: `hover:shadow-md` — слабый эффект.
- Контент: `p-3` — тесно, иерархия «название → цена → мета» слабая.
- Нет индикатора «количество фото» на карточке.
- Цена `text-lg` — ок, но могла бы выделяться сильнее.

**Рекомендации:**
- Добавить `shadow-sm` по умолчанию, `shadow-lg` на hover.
- Увеличить padding контентной области.
- Бейджи: чуть крупнее, иконки чётче.
- Добавить счётчик фото (e.g. «5 фото») в угол изображения.

---

## 5. Trust-signals

**Проблемы:**
- «Проверено», «Один владелец», «Хорошая цена» — есть, но визуально слабые.
- В SellerCard: CheckCircle и «Документы подтверждены» — неплохо, но блок не выделен.
- VIN report: иконка ShieldCheck ок, но блок выглядит как «просто форма».
- Нет глобальных trust-elements в header/footer (защита сделки, гарантии).

**Рекомендации:**
- Усилить бейджи на карточках: чуть крупнее, иконки заметнее.
- Добавить trust-bar в footer: «Безопасные сделки», «Проверка VIN», «Гарантия».
- Listing: выделить VIN/Seller блоки визуально (background, border).

---

## 6. Header

**Проблемы:**
- `h-14` — стандартно, но не «premium».
- Логотип 8×8 — мелкий, «АС» аббревиатура не запоминается визуально.
- Навигация: только текст, без иконок — плоский вид.
- Ghost-кнопки: `text-muted-foreground` — визуально «уходят».
- CTA «Подать объявление» — один primary, но окружение слабое.
- Нет подложки/тени при scroll.

**Рекомендации:**
- Увеличить высоту до `h-16`, добавить лёгкую тень при скролле.
- Логотип чуть крупнее, возможно слот для словесного знака.
- Добавить subtle background/blur при scroll.
- Иконки в навигации (опционально).
- Более чёткий контраст между ghost и primary.

---

## 7. Search panel

**Проблемы:**
- Region chips: `rounded-full` — ок, но `hover:bg-gray-100` — хардкод, не token.
- Condition tabs: тонкий индикатор `h-0.5` — слабо видно.
- Filter panel: `shadow-sm` — почти не видно.
- FilterSelect: `absolute -top-2` label — типичный floating label, но с `bg-white` обрезается на границах.
- RangeInput: два input в ряд — без визуального разделителя между «от» и «до».
- Кнопка «Показать объявления» — `lg`, но панель в целом «простая».

**Рекомендации:**
- Панель: усилить shadow, возможно `ring-1` для чёткости.
- Chips: использовать design tokens для hover.
- Tabs: более выразительный индикатор (высота, цвет).
- Добавить разделители между группами фильтров.
- Trust-подпись: «Тысячи проверенных объявлений» под поиском.

---

## 8. Listing page

**Проблемы:**
- Hero: заголовок `text-2xl` — слабый для страницы объявления.
- Breadcrumb: простые span, без компонента.
- Галерея: ок, но блок не имеет визуального «фрейма».
- Price block: `text-3xl` — неплохо; credit block — `bg-surface-info` ок.
- Price status (good/normal/high): `bg-green-50` и т.д. — хардкод, не tokens.
- Seller block: ровный, но без «premium» ощущения.
- Similar cars: тот же VehicleCard — норм.

**Рекомендации:**
- Hero: более крупный заголовок, возможно подзаголовок (год, город).
- Price: использовать design tokens для status.
- Seller: выделить карточку (border, subtle shadow).
- Gallery: рамка/фон для «премиальности».

---

## 9. Мобильная версия

**Проблемы:**
- Header: много скрытых элементов (`hidden sm:inline`, `hidden md:flex`) — пустой вид на мобиле.
- Search: фильтры в колонку — длинный скролл.
- Карточки: `grid-cols-1` — ок.
- Listing: колонки штабелируются — нормально, но порядок блоков можно оптимизировать.
- Touch targets: кнопки 8×8 (Heart) — на границе 44px.

**Рекомендации:**
- Убедиться touch targets ≥ 44px.
- Mobile header: hamburger или приоритетные действия.
- Search: consider collapsible filters на мобиле.

---

## 10. Dark theme

**Проблемы:**
- VehiclePrice: `bg-green-50`, `text-green-700` — светлые, в dark не адаптируются.
- Search: `bg-white` в FilterSelect — хардкод.
- Карточки: `bg-card` ок, но некоторые элементы (gray-50, gray-100) жёсткие.

**Рекомендации:**
- Заменить все `gray-*`, `green-*`, `blue-*`, `orange-*` на semantic tokens.
- Проверить все компоненты в dark mode.

---

## 11. Motion и microinteractions

**Проблемы:**
- Почти нет motion: только `transition-colors`, `transition-all`.
- Нет staggered reveal для карточек.
- Hover на карточках — только shadow.
- Кнопки — стандартный transition.
- Framer Motion в deps, но не используется в marketplace/listing.

**Рекомендации:**
- Staggered fade-in для recommendations при загрузке.
- Hover: scale или более плавный shadow transition.
- Кнопки: subtle scale на active.
- Listing gallery: плавные transitions уже есть — усилить.

---

## 12. Бренд и ощущение

**Проблемы:**
- Teal palette есть, но применяется точечно.
- Нет «фирменного» визуального элемента (pattern, gradient, форма).
- Footer минималистичен — возможность для trust/links.
- В целом «чисто», но не «запоминаемо».

**Рекомендации:**
- Ввести акцентный gradient или subtle pattern для hero-зон.
- Использовать teal более системно (не только кнопки).
- Добавить «фирменный» элемент в header или между секциями.
